\section{Conclusion}
\label{sec:conc}
In this paper, a spiking transformer attention model called obSTAtten was developed from the model STAtten\cite{STAtten}, which aimed to allow for overlapping temporal blocks while still utilizing the mathematical framework of STAtten. This required making modifications to STAtten, resulted in key findings, and has produced two interesting avenues for future work. This paper has some limitations which are also mentioned below.

\subsection{Summary of Contribution}
Producing an attention model obSTAtten from the STAtten code-base required implementing three major features, as detailed below.

\subsubsection{Determining Overlapping-Block Count}
A new and slightly more complex approach to determining the number of temporal blocks ($N_B$) was developed to allow overlapping blocks and shared timesteps. This was implemented for a stride $s=1$ and an overlap $T_{ov}=1$ for simplicity and reliability in testing.

\subsubsection{Overlapping Block Construction}
Next $N_B$ overlapping blocks $\mathcal{B}_i$ for index $i$ are generated of the form $[i:i=N_B,:,d]$. These blocks are the same shape as the blocks generated by STAtten allowing the rest of existing framework to utilize these blocks with no extra modification.

\subsubsection{Temporal Output Averaging}
The model output $\hat{Y}_t$ for timestep $t$ requires the contributions of each timestep $t$ to be averaged before they are utilized. This step does not apply to the first or last timestep $t=0$ or $t=T-1$ as these timesteps only exist on one block.

\subsection{Key Findings}
There were several notable findings that resulted from the experiment detailed in this paper, the details of each one will be expanded on below.

\subsubsection{Model Complexity Effect on obSTAtten Performance}
A model's complexity can be derived from the dimension of its trainable parameters (256 or 512) and by the presence of parameterization of the membrane time potential $\tau$ (LIF or PLIF). Models that lack sufficient complexity do not seem to benefit from obSTAtten and they do not significantly benefit from STAtten either when using static image inputs (CIFAR10).

\subsubsection{LIF vs PLIF Effect on obSTAtten Performance}
PLIF neurons have learnable membrane time constant $\tau$ allowing them to learn a temporal parameter. This feature already allows PLIF neurons to correlate temporal information, as such the extra temporal information provided by obSTAtten is either redundant, confusing or noise inducing which lowers the test accuracy of the model.

\subsubsection{Training Times per Epoch}
When STAtten and obSTAtten were trained on CIFAR10 for the conditions 512-dim and LIF neurons, obSTAtten outperforms STAtten. During this experiment it can be seen around epoch 140 that obSTAtten becomes slightly slower-to-train than STAtten, but it catches back up. It is only significantly faster at training before this event.

\subsubsection{256-dim LIF Convergence}
When the least complex model was trained, the STAtten and obSTAtten model behavior is identical, meaning for models of low enough complexity STAtten and obSTAtten are computationally equivalent (but not equal in training speed). This was run twice to ensure it was not an error.

\subsubsection{STAtten vs obSTAtten for Best-Epoch}
STAtten seems to take more epochs to reach its best epoch than obSTAtten does in most cases, suggesting the improved temporal context does improve early learning rates when adjustments to the network are coarser. This effect settles off as the model matures towards epoch 210.


\subsection{Limitations}
Due to hardware constraints and time constraints, only CIFAR10 was used in training, as such none of the experiments were run using DVS or other dynamic inputs. It is possible that such dynamic input would have made the two models behave in ways not replicatable using static image input.

\subsection{Future Work}
Future work can take two possible avenues, the first would be a deeper dive into the current state of obSTAtten by allowing more hyper-parameter tuning than the model currently does. The second option would have obSTAtten train on different datasets such as CIFAR100\cite{cifar10} and CIFAR10-DVS. 

\subsubsection{Stride and Overlap Adjustment}
In the current implementation of obSTAtten, stride and overlap for temporal blocks are both hard-coded values, adjusting them may yield interesting results and should be explored in future work.

\subsubsection{CIFAR100 and CIFAR10-DVS}
Currently hardware and storage space limitations prevent the usage of larger datasets like CIFAR10-DVS which may yield interesting results that static datasets such as CIFAR10 or CIFAR100 do not. This may be useful to study for obSTAtten because SNNs are temporally dynamic unlike ANNs.
